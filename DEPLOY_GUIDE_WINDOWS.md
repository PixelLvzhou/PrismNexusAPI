## 1. 服务器配置准备

### ❓ 你的疑问解答
- **本地有Node.js，服务器需要安装吗？**
  → **需要！** 本地和服务器是两个不同的环境

### 第一步：服务器上安装Node.js

1. 在服务器上打开浏览器，访问：
   https://nodejs.org/

2. 下载 **LTS版本**（长期支持版）

3. 双击安装包，一路"Next"安装

4. 验证安装：
   打开 **CMD**（命令提示符）输入：
   ```cmd
   node --version
   npm --version
   ```
   看到版本号就成功了

## 2. 后端部署步骤

### 第一步：把项目传到服务器

有3种方式，选你最习惯的：

#### 方式A：直接复制粘贴（最简单）
1. 在你电脑上找到 `c:\Users\lenovo\Desktop\个人项目\backend`
2. 右键 → 压缩为 ZIP文件
3. 通过远程桌面复制到服务器
4. 在服务器上解压到 `D:\backend\`

#### 方式B：使用Git（如果你有Git仓库）
```cmd
cd D:\
git clone 你的仓库地址.git
```

#### 方式C：使用FTP工具
使用 FileZilla 等工具上传到服务器

---

### 第二步：在服务器上安装依赖

1. 打开服务器的 **CMD**，进入项目目录：
   ```cmd
   d:
   cd backend
   ```

2. 安装依赖：
   ```cmd
   npm install
   ```

---

### 第三步：配置环境变量

在服务器的 `D:\backend\` 目录下，创建一个新文件，命名为 `.env`

**编辑 `.env` 文件内容：**
```env
# QQ邮箱配置
SMTP_HOST=smtp.qq.com
SMTP_PORT=465
SMTP_USER=你的QQ邮箱@qq.com
SMTP_PASS=你的QQ邮箱SMTP授权码

# 服务配置
PORT=3000

# JWT密钥（自己随便写个32位字符串就行）
JWT_SECRET=写个你喜欢的32位随机字符串
JWT_EXPIRES_IN=24h
```

---

### 第四步：启动后端服务

#### 方式1：直接启动（临时测试）
```cmd
node server.js
```
看到 "服务运行在 http://localhost:3000" 就可以了

#### 方式2：使用PM2（推荐，生产环境用）
```cmd
# 安装PM2
npm install -g pm2

# 启动
pm2 start server.js --name prism-nexus-backend

# 查看状态
pm2 status

# 看日志
pm2 logs prism-nexus-backend
```

## 3. 前端部署方案

### ❓ 选择建议

| 方案 | 优点 | 缺点 |
|------|------|------|
| **继续用GitHub Pages** | 免费，已部署 | 速度可能不如国内，后端跨域需配置 |
| **迁移到阿里云** | 国内访问快，前后端同域 | 需要自己配置服务器 |

---

### 方案A：继续用GitHub Pages（推荐先试试）

只需要改一下前端的API地址：

在你的前端项目 `PrismNexus` 里：

1. 创建文件：`.env.production`
2. 内容：
   ```env
   VITE_API_BASE_URL=http://你的阿里云服务器IP:3000
   ```
3. 重新构建并部署到GitHub

⚠️ **注意：需要在阿里云安全组开放3000端口！**

---

### 方案B：迁移到阿里云（前后端一起）

#### 第一步：把前端传到服务器
同样复制 `PrismNexus` 文件夹到服务器，比如 `D:\prism-nexus\`

#### 第二步：在服务器上构建前端
```cmd
cd D:\prism-nexus
npm install
npm run build
```

#### 第三步：部署前端（二选一）

**方式1：使用简单的Node静态服务器**
```cmd
cd D:\prism-nexus\dist
npx serve -l 80
```
（需要管理员权限）

**方式2：安装IIS（Windows服务器自带Web服务）**
1. 服务器管理器 → 添加角色和功能
2. 勾选Web服务器(IIS)
3. 把 `dist` 文件夹内容放到 `C:\inetpub\wwwroot\`
4. 浏览器访问 `http://服务器IP` 就能看到

## 4. 安全配置（必做）

### 第一步：开放服务器端口

在阿里云控制台操作：

1. 登录阿里云 → 云服务器ECS → 网络与安全 → 安全组
2. 找到你的实例对应的安全组，点击"配置规则"
3. 添加规则：

| 端口范围 | 授权对象 | 说明 |
|----------|----------|------|
| 3000 | 0.0.0.0/0 | 后端API |
| 80 | 0.0.0.0/0 | 前端HTTP（可选） |
| 443 | 0.0.0.0/0 | 前端HTTPS（可选） |

---

### 第二步：Windows防火墙

在服务器上：
1. 打开"高级安全Windows Defender防火墙"
2. 入站规则 → 新建规则
3. 端口 → TCP → 特定本地端口 → 3000
4. 允许连接
5. 全部勾选 → 名称：后端API

---

### 第三步：验证一切正常

在你本地电脑浏览器访问：
```
http://你的阿里云服务器IP:3000/health
```
看到 "ok" 就说明后端部署成功了！
